# Contributing

Thank you for your interest in contributing to ReqNexa AI!

## Getting Started
- Fork the repository and create a feature branch from `develop`.
- Keep PRs focused and scoped; prefer small, incremental changes.

## Development Workflow
- Install dependencies in each workspace: `backend/` and `frontend/`.
- Run locally:
  - Backend: `npm run dev` in `backend/`
  - Frontend: `npm run dev` in `frontend/`
- Write tests for new code and ensure existing tests pass.

## Code Style
- Follow existing patterns and conventions in the codebase.
- Keep changes minimal and secure; do not log secrets.

## Commits & PRs
- Use clear, descriptive commit messages.
- Reference issues in PR descriptions where applicable.
- Ensure CI passes (build and tests) before requesting review.
